# 功能规格（Feature Specs）

本文件收纳较“可落地”的实施说明与验收要点。算法细节请以对应算法文档为准，不在此重复维护公式。

## 1) 阈值工具（待实现）

### 目标
- 在工具页新增“阈值工具”入口，提供游泳/骑行/跑步三类阈值与分区计算
- 交互与样式尽量复用现有 `pages/tools/` 的结构与视觉规范

### 范围与页面结构
- 入口：`pages/tools/tools` 增加一个 Tab 或入口按钮
- 子页：建议独立成 `pages/tools/threshold/*`（swim/bike/run）以保持页面清晰
- 每个子页统一四区块：标题区、输入区、操作区（计算/重置）、结果区（卡片列表）

### 算法来源与单一权威
- 游泳 CSS、骑行 FTP 分区、跑步阈值/分区与预测：请以 训练助手-算法说明.md 为权威来源
- 若阈值工具与训练助手使用同一套算法：应抽到公共工具（如 `utils/thresholdUtils.js`），避免两处计算逻辑分叉

### 验收要点
- 输入校验：必填项、时间格式、边界条件（例如游泳 t400 > t200）能提示并阻止计算
- 输出格式：单位统一（游泳 `/100m`、骑行 `W`、跑步 `min/km`），区间边界四舍五入一致
- 结果稳定：用 3 组以上样例验证输出（正常值、边界值、异常值）

## 2) 打卡积分规则调整（待确认/可能已过期）

### 背景
- 旧规格中提到“青少年短距离”积分从 80 调整为 200，但当前 `pages/upload/upload.js` 的比赛积分映射并不存在该条目（当前使用 `大铁/70.3/标铁/路跑赛事（半程以上）`）

### 建议处理方式
- 若该需求仍有效：先明确“青少年短距离”在当前产品中的入口（比赛类型？活动类型？）
- 明确后再更新对应 `pointMap` 与规则文案，并补齐预览/提交一致性校验

### 验收要点
- 前端预览分值与提交到后端的 `points` 一致
- 历史记录展示与统计口径不被破坏（排行榜/个人统计）

## 3) 服装发放：复制快递信息（已完成）

### 功能
- 在服装发放管理卡片操作区新增“复制”，一次复制姓名/手机号/地址到剪贴板

### 验收要点
- 姓名/手机/地址为空时给出明确提示，不写入空内容
- 复制文本格式为两行：`姓名 手机号` 与 `地址`
