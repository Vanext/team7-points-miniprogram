# 项目开发文档索引

本目录用于沉淀“团队7积分小程序”的开发过程、现状架构、路线图与专题说明，避免同一内容在多份文档重复维护。

## 推荐阅读顺序

1. CHANGELOG.md：已经做了什么（按版本/里程碑汇总）
2. ARCHITECTURE.md：系统现在长什么样（以代码库现状为准）
3. ROADMAP.md：接下来要做什么（目标与待办，弱化排期）
4. FEATURE-SPECS.md：功能规格与实施要点（面向落地）
5. 训练助手-算法说明.md：训练助手相关算法与验证（权威来源）
6. ALGORITHMS.md：算法目录（索引与变更约定）

## 文件说明

| 文档 | 用途 | 更新频率 | 适用读者 |
|---|---|---:|---|
| CHANGELOG.md | 变更日志：功能合入与修复汇总、影响面、部署要点 | 每次合入/发布 | 全员 |
| ARCHITECTURE.md | 现状架构：关键模块、数据模型、核心云函数与调用链 | 架构/数据模型变更时 | 开发/维护 |
| ROADMAP.md | 路线图：未完成目标与改进项（Backlog） | 需求/规划调整时 | 产品/开发 |
| FEATURE-SPECS.md | 功能规格：较完整的实施方案、验收要点、边界条件 | 做功能前/后补齐 | 开发/测试 |
| 训练助手-算法说明.md | 算法说明：训练助手阈值与分区、预测算法 | 算法改动时 | 开发/运营 |
| ALGORITHMS.md | 算法索引：汇总算法文档入口与“单一权威来源”约定 | 很少 | 开发 |

## 文档维护约定

- 单一事实来源：同一算法/规则只在一份文档维护，其它地方只做链接或引用摘要。
- CHANGELOG 只写“结论与影响”，不要再塞长篇实现细节与示例代码。
- ARCHITECTURE 只描述“已存在的实现”，规划内容放 ROADMAP 或 FEATURE-SPECS。
- 旧文档历史内容统一放到 ARCHIVE/，默认不再更新。
